# Symmetric Encryption and Key Derivation (PBKDF2 + Fernet) in Python

This repository contains a Python script demonstrating how to securely encrypt and decrypt messages using **PBKDF2** (Password-Based Key Derivation Function 2) for key derivation and **Fernet** (AES-128 in CBC mode with HMAC authentication) from the `cryptography` library.

## 📌 Project Overview

When encrypting sensitive data, hardcoding raw encryption keys can be risky and inconvenient. Instead, key derivation functions like PBKDF2HMAC transform user-supplied passwords into cryptographically strong keys.

This script demonstrates a full cryptographic workflow:

1. **Key Derivation:** Derives a 256-bit key from a password and salt using PBKDF2 with SHA-256 (100,000 iterations).
2. **Key Formatting:** Base64-encodes the derived key into the format required by Fernet.
3. **Encryption:** Encrypts a plaintext message into a secure, authenticated ciphertext.
4. **Decryption:** Decrypts the ciphertext back into its original plaintext.

---

## 📖 Step-by-Step Code Explanation

### Step 1: Importing Cryptographic Libraries

The script imports `base64` and `os` from Python's standard library, alongside key derivation (`PBKDF2HMAC`), hashing (`hashes`), default backend functions, and the `Fernet` symmetric encryption recipe from the `cryptography` module.

```python
import base64
import os
from cryptography.hazmat.backends import default_backend
from cryptography.hazmat.primitives import hashes
from cryptography.hazmat.primitives.kdf.pbkdf2 import PBKDF2HMAC
from cryptography.fernet import Fernet

```

### Step 2: Unused Key Generation (Overridden later)

The script generates 32 cryptographically secure random bytes using `os.urandom(32)`.

```python
# Generate a random 256-bit key
key = os.urandom(32)

```

> 💡 **Code Note:** This initial `key` variable is immediately overwritten in Step 3 when the derived key is assigned to it.

### Step 3: Deriving a Secure Key from a Password

The code sets up a byte-string `password` and `salt`. It configures PBKDF2HMAC using SHA-256 as its underlying hash function, requesting a output length of 32 bytes ($256$ bits) and forcing $100,000$ iterations to protect against brute-force dictionary attacks.

```python
# Derive a key from the password
password = b"my_secret_password"
salt = b"my_secret_salt"
kdf = PBKDF2HMAC(
    algorithm=hashes.SHA256,
    length=32,
    salt=salt,
    iterations=100000,
    backend=default_backend()
)
key = base64.urlsafe_b64encode(kdf.derive(password))

```

* `.derive(password)` processes the password through the key derivation algorithm.
* `base64.urlsafe_b64encode(...)` encodes the raw 32-byte binary key into a URL-safe Base64 string format required by `Fernet`.

### Step 4: Creating the Fernet Cipher Instance

A `Fernet` cipher object is created using the Base64-encoded key. Fernet provides symmetric encryption guarantee: data encrypted with it cannot be manipulated or read without the exact derived key.

```python
# Create a Fernet object using the derived key
fernet = Fernet(key)

```

### Step 5: Encrypting the Plaintext Message

The original message (`plaintext`) is passed to `fernet.encrypt()`. This generates an authenticated, encrypted ciphertext containing a timestamp, an initialization vector (IV), the AES-128-CBC encrypted payload, and an HMAC signature.

```python
# Encrypt the plaintext
plaintext = b"This is my secret message"
ciphertext = fernet.encrypt(plaintext)
print(ciphertext)

```

### Step 6: Decrypting the Ciphertext

Finally, `fernet.decrypt()` validates the HMAC signature to verify authenticity and decrypts the ciphertext, returning the original plaintext byte-string.

```python
# Decrypt the ciphertext
plaintext = fernet.decrypt(ciphertext)
print(plaintext)

```
